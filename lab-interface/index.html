Docker Workshop Lab Tutorial

Interactive hands-on Docker learning experience

🚀 Getting Started with Docker

Welcome to this hands-on Docker tutorial! You'll learn how to work with containers, images, and multi-container applications using a real development environment.

🎯 What You'll Learn

  • Docker fundamentals and container lifecycle
  • Building and managing container images
  • Working with volumes and networking
  • Multi-container applications with Docker Compose
  • Best practices and debugging techniques

💡 How to Copy Commands

Method 1: Click any code block to copy commands instantly. Your browser may ask for clipboard permission - please allow it for the best experience.

Method 2: If clipboard access is blocked, manually select the code text and use Ctrl+C (or Cmd+C on Mac) to copy, then Ctrl+V to paste in the VS Code terminal.

Tip: You can also press Enter or Space when a code block is focused.

1
Verify Your Environment

First, let's make sure Docker is properly installed and working in your environment.

docker --version
docker info

You should see Docker version information and system details.

2
Check Running Containers

See what containers are currently running in your isolated environment.

docker ps
docker ps -a

⚠️ Isolated Environment

Your environment is isolated for security. You'll only see containers created within this workshop.

3
Your First Container

Let's run your first container to test the setup.

docker run hello-world

This downloads and runs a test container that displays a welcome message.

📦 Working with Containers

Learn how to create, manage, and interact with Docker containers.

4
Interactive Containers

Run an interactive container where you can execute commands.

docker run -it --name my-ubuntu ubuntu bash

Inside the container, try these commands one by one:

whoami
pwd
ls -la
cat /etc/os-release
echo "Hello from inside the container!"
exit
5
Container Lifecycle

Learn how to manage container lifecycle with start, stop, and remove operations.

# See all containers (running and stopped) docker ps -a
# Start a stopped container docker start my-ubuntu
# Stop a running container docker stop my-ubuntu
# Remove a container docker rm my-ubuntu
6
Running Web Services

Start a web server and learn about port mapping.

docker run -d --name webserver -p 8080:80 nginx:alpine

Test the web server:

curl http://localhost:8080
docker logs webserver

✅ Port Forwarding

The workshop environment automatically forwards ports, so you can access services via localhost.

🏗️ Images and Building

Learn how to work with Docker images and build your own.

7
Exploring Images

Understand Docker images and how to manage them.

# List local images docker images
# Pull a specific image docker pull node:18-alpine
# Inspect an image docker inspect node:18-alpine
# See image layers docker history node:18-alpine
8
Building Your First Image

Create a simple Dockerfile and build a custom image.

# Create a simple Dockerfile cat > Dockerfile << 'EOF' FROM node:18-alpine WORKDIR /app COPY package*.json ./ RUN npm install COPY . . EXPOSE 3000 CMD ["npm", "start"] EOF

Build the image:

docker build -t my-app:latest .

🔗 Multi-Container Applications

Learn to orchestrate multiple containers using Docker Compose.

9
Docker Compose Basics

Start the multi-container application stack.

# View the compose configuration cat docker-compose.yml
# Start all services docker compose up -d
# Check service status docker compose ps
10
Service Communication

Learn how containers communicate in a compose network.

# Execute commands in running containers docker compose exec workspace bash

Inside the container, test connectivity:

ping db
nc -z redis 6379

View service logs:

# View service logs docker compose logs -f webapp

🤖 Agentic Compose - Building AI Agents with Docker

Learn how to build, ship, and run AI agents using Docker Compose. This workshop covers Docker's new AI features including Model Runner and MCP Gateway for creating intelligent agentic applications.

🎯 What You'll Build

Agentic applications need three things: models, tools, and code that glues them together. This workshop walks through each component using Docker's latest AI capabilities.

  • 🧠 Models: Docker Model Runner for local AI inference
  • 🛠️ Tools: MCP (Model Context Protocol) Gateway for external tool integration
  • 🧑‍💻 Code: Frameworks like LangGraph, CrewAI, Spring AI integration

⚠️ Prerequisites

  • Docker Desktop 4.43.1+ (easiest) or Docker Engine 28.3+ with Compose 2.38.1+
  • Docker Model Runner 0.1.32+
  • Basic understanding of AI concepts (models, agents, tools)
13
Verify Docker AI Prerequisites

First, ensure you have Docker's AI features properly installed and running.

docker model status
docker compose version
docker model list

You should see Docker Model Runner is active and ready to run AI models locally.

14
Clone the Agentic Compose Workshop

Get the workshop materials and explore the structure.

git clone https://github.com/ajeetraina/workshop-agentic-compose.git
cd workshop-agentic-compose
ls -la
cat compose.yaml
15
Understanding Models - Docker Model Runner

Learn about running AI models locally with Docker Model Runner.

Pull a language model (this may take a while - ~2.9GB):

docker model pull llama3.2:3b

Check model details:

docker model inspect llama3.2:3b

Start the model server:

docker model serve llama3.2:3b --port 11434

🧠 About Docker Model Runner

Docker Model Runner provides a consistent way to run AI models locally, supporting various model formats and providing a unified API for inference across different model types.

16
Testing Model Inference

Test your local model with a simple inference request.

Test the model with a chat completion:

curl -X POST http://localhost:11434/v1/chat/completions \ -H "Content-Type: application/json" \ -d '{ "model": "llama3.2:3b", "messages": [ {"role": "user", "content": "What is Docker and why is it useful?"} ] }'

Or test with a simpler endpoint:

curl http://localhost:11434/api/generate \ -d '{"model": "llama3.2:3b", "prompt": "Explain containerization briefly:"}'
17
Setting Up MCP Gateway

Configure the Model Context Protocol Gateway for tool integration.

docker compose up mcp-gateway -d
docker compose ps mcp-gateway
curl http://localhost:3001/mcp/tools

🛠️ About MCP Gateway

The MCP Gateway provides a secure, standardized way for AI agents to connect to external tools and services. It acts as a bridge between your AI models and various APIs, databases, and other tools.

18
Compose Configuration for AI Services

Examine how to configure AI services in Docker Compose.

# View the complete compose configuration cat compose.yaml
# Start all services in the background docker compose up -d
# Monitor service startup docker compose logs -f
# Check all services are running docker compose ps

Your compose file defines models, MCP tools, and application services working together.

19
Building Your First AI Agent

Create a simple AI agent that uses both the model and MCP tools.

# Access the agent development environment docker compose exec agent-dev bash

Inside the agent container, explore the code:

# View the agent code structure ls -la /app
# Look at the main agent script cat /app/simple_agent.py
# Run your first agent python /app/simple_agent.py
20
Integrating Popular Frameworks

Learn how to integrate with popular agentic frameworks using Compose.

# Start LangGraph service docker compose up langgraph -d
# Start CrewAI service docker compose up crewai -d
# Test framework integration curl http://localhost:8001/langgraph/status
# View framework logs docker compose logs langgraph

🔗 Framework Integration

Docker Compose seamlessly integrates with popular frameworks:

  • LangGraph: For building stateful, multi-actor applications
  • CrewAI: For multi-agent collaboration
  • Spring AI: For Java-based AI applications
  • Vercel AI SDK: For TypeScript/JavaScript agents
21
Working with MCP Tools

Explore and use various MCP tools for extending agent capabilities.

# List available MCP tools docker run --rm docker/mcp-toolkit list
# Install a file system MCP tool docker run --rm docker/mcp-toolkit install filesystem
# Install a web search MCP tool docker run --rm docker/mcp-toolkit install web-search
# Test tool connectivity curl http://localhost:3001/mcp/tools/filesystem/status
22
Agent with Tool Integration

Build an agent that uses multiple tools through the MCP Gateway.

# Run the advanced agent with tools docker compose exec agent-dev python /app/tool_agent.py
# Monitor tool usage in MCP Gateway docker compose logs mcp-gateway -f
# Test agent capabilities curl -X POST http://localhost:8002/agent/task \ -H "Content-Type: application/json" \ -d '{"task": "Search for Docker documentation and summarize key points"}'
23
Production Deployment

Prepare your agentic application for production deployment.

# Build production images docker compose build --parallel
# Tag images for deployment docker tag workshop-agent:latest myregistry/workshop-agent:v1.0
# View production compose configuration cat compose.prod.yaml
# Deploy to production environment docker compose -f compose.prod.yaml up -d

🚀 Production Ready

Your agentic application is now containerized and ready for deployment with proper scaling, monitoring, and security configurations.

🚀 Advanced Topics

Explore advanced Docker concepts and debugging techniques.

11
Volume Management

Work with persistent data using volumes.

# Create a volume docker volume create my-data
# Run container with volume docker run -it --rm -v my-data:/data ubuntu bash

Inside container:

echo "Persistent data!" > /data/test.txt
exit

Verify data persists:

# Verify data persists docker run -it --rm -v my-data:/data ubuntu cat /data/test.txt
12
Debugging Containers

Learn debugging techniques for containerized applications.

# Inspect running containers docker inspect [container-name]
# View resource usage docker stats [container-name]
# Debug inside containers docker exec -it [container-name] sh
# View container processes docker top [container-name]
